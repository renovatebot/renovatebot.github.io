If you're using the GitHub-hosted app version of Renovate, and you invited the renovate npm user into your npm organization then keep reading this message.

We are phasing out using the renovate npm user for private packages. We are deliberately skipping provisioning this npm token for 20 percent of the time (brownout). This means that Renovate will autoclose some pull requests, which will then be re-opened the next time Renovate runs with the token.

If you're seeing this behavior, it means you need to set up your own service user token. Read discussion 20613 on the Renovate repository to learn more.
